Git команды
Git команды
git -v - показать версию Git
git init - создать репозиторий в текущей папке
git clone [remote] - клонировать репозиторий [remote] в текущую папку
git add - добавить [element] в индекс (staging area) для последующего коммита
git status - показать состояние файлов
git status -s - показать состояние файлов (краткий вид)
git reset [commit] - откат файлов к состоянию [commit]
git rm --cached [element] - отменить слежение за [element]
git status - показать статус локального репозитория
git status -s - показать статус локального репозитория в краткой форме

Remote
git remote -v - показать адрес привязанного удаленного репозитория
git remote add [name] [url] - добавить удаленный репозитрий с именем [name]
git remote rename [old] [new] - переименовать удаленный репозиторий [old] в [new]
git remote remove [name] - отключить удаленный репозиторий [name]

Branch
git branch [branch] - создать ветку [branch]
git checkout [branch] - переключить на ветку [branch]
git checkout -b [branch] - создать ветку [branch] и переключить на нее
git branch - показать все ветки
git branch -m [old] [new] - переименовать ветку [old] в [new]
git branch -vv - посмотреть связи локальных веток в удаленными
- - -
git switch main [branch] - переключить на ветку [branch]
git switch -c [branch] - создать ветку [branch] и переключить на нее
git switch - - вернуться на предыдущую ветку
git branch -d [branch] - удалить ветку [branch]
git merge [branch] - слить ветку [branch] с текущей
git branch - показать список веток

Log
git log - показать истории коммитов
git log --oneline - показать истории в однострочном формате
git log --stat - показать истории с показом списка измененных файлов
git log -n [n] - показать истории с показом [n] последних коммитов
git log --pretty=[preset] - показать статуса локального репозитория в формате [preset]

Commit
git commit - записать выбранные файлы в репозиторий
git commit -m [comment] - записать выбранные файлы в репозиторий с комментарием [comment]
git commit -a - записать выбранные файлы в репозиторий пропуская индексацию

Push:
git push [remote] [branch] - отправить в [remote] ветку [branch]
git push --set-upstream [remote] [branch] - отправить в [remote] ветку [branch] с запоминанием опций
git push -u [remote] [branch] - отправить в [remote] ветку [branch] с запоминанием опций
git push - отправить данные согласно запомненных опций
